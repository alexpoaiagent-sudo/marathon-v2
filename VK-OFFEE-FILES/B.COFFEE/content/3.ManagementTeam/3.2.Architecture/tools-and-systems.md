# F8: Инструменты и системы

**Семейство:** F8 (Система создания × Инженер)
**Вопрос:** Какие инструменты используются для создания продукта?

## Назначение документа

Этот документ описывает **инструменты, оборудование и системы**, которые использует команда для создания и поддержания работы сети кофеен.

## Операционные системы

### POS (Point-of-Sale) — Касса

**Назначение:** Приём заказов, оплата, передача в производство.

**Функции:**
- Пробитие чеков
- Приём оплаты (наличные, карты)
- Передача заказов в KDS
- Учёт выручки
- Отчёты по сменам

**Требования:**
- Скорость: заказ в работу ≤ 10 сек
- Надёжность: аптайм > 99%
- Интеграция с KDS
- Интеграция с эквайрингом

**SLA:**
- Передача заказа в KDS: ≤ 10 сек после оплаты
- Время отклика: < 1 сек

### KDS (Kitchen Display System) — Экран заказов

**Назначение:** Отображение заказов для бара и кухни.

**Функции:**
- Приём заказов с POS
- Отображение приоритетов
- Таймер времени выполнения
- Статусы заказов (в работе/готово)
- Уведомления о задержках

**Требования:**
- Видимость экрана с рабочего места
- Цветовая индикация (зелёный/жёлтый/красный)
- Звуковые уведомления
- Синхронизация между баром и кухней

**SLA:**
- Получение заказа с POS: ≤ 1 сек
- Обновление статуса: реал-тайм

### Учётная система

**Назначение:** Складской учёт, инвентаризации, списания.

**Функции:**
- Учёт поступлений от поставщиков
- Списание по рецептурам
- Инвентаризации
- Контроль остатков
- FIFO (First In — First Out)

**Требования:**
- Интеграция с POS
- Автоматическое списание
- Контроль критических остатков
- Отчёты по фудкосту

## Процессы снабжения

### 1. Планирование потребности

**Метод:** На основе прогноза продаж

**Формула:**
```
Потребность = Прогноз продаж × Норма расхода + Страховой запас
```

**Пример (кофе):**
- Прогноз: 200 чашек/день
- Норма: 18г/чашку
- Потребность: 200 × 18г = 3.6 кг/день
- На неделю: 3.6 × 7 = 25.2 кг
- Страховой запас: +20% = 30 кг

### 2. Заказ

**Процесс:**
1. Проверка остатков (ежедневно)
2. Формирование заявки (при достижении минимума)
3. Отправка заявки поставщику
4. Подтверждение от поставщика
5. Контроль доставки

**Минимальные остатки (par-уровни):**
- Кофе: 10 кг
- Молоко: 20 л
- Выпечка: запас на 1 день

### 3. Приёмка

**Процесс:**
1. Проверка соответствия заявке
2. Проверка качества (внешний вид, запах, срок годности)
3. Проверка количества
4. Оформление накладной
5. Внесение в учётную систему

**Критерии отказа:**
- Несоответствие заявке
- Истёкший/близкий срок годности
- Повреждённая упаковка
- Несоответствие качеству

### 4. Хранение

**Принцип FIFO:** First In — First Out (первым пришёл — первым ушёл)

**Температурные режимы:**
- Молоко: 2-6°C
- Заморозка: -18°C
- Сухие продукты: комнатная температура
- Кофе: комнатная температура, герметичная упаковка

**Контроль:**
- Ежедневная проверка сроков годности
- Ротация запасов по FIFO
- Контроль температуры холодильников

## Инфраструктура

### Электричество

**Требования:**
- Мощность: 15-20 кВт на точку
- Стабильность: без перепадов
- Резервное питание: ИБП для POS и KDS

**Основные потребители:**
- Кофемашина: 3-4 кВт
- Духовка: 2-3 кВт
- Холодильники: 1-2 кВт
- Кондиционер: 2-3 кВт
- Освещение: 1-2 кВт

### Вода

**Требования:**
- Качество: фильтрованная
- Давление: стабильное
- Температура: холодная + горячая

**Фильтрация:**
- Предфильтр (механическая очистка)
- Угольный фильтр (хлор, запахи)
- Умягчитель (для кофемашины)

**Замена фильтров:** раз в 3-6 месяцев

### Интернет

**Требования:**
- Скорость: > 50 Мбит/с
- Стабильность: аптайм > 99%
- Wi-Fi для гостей: отдельная сеть

**Использование:**
- POS и KDS
- Учётная система
- Wi-Fi для гостей
- Музыкальный стриминг
