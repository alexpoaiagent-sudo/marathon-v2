# CODEX — правила базы знаний

- Используем `kebab-case` в латинице без пробелов, скобок и кириллицы в путях.
- Для новых файлов добавляем ссылки в профильный `README.md`, «Навигацию по базе проекта» (`/ru/`) и `knowledge-inventory.md`.
- Не храним рабочие копии в корне: распределяем материалы по целевым разделам (`ops/`, `training/`, `vendors/`, `finance/`, `assets/`, `system/`, `franchise/`).
- Бинарные файлы (docx, xls/xlsx, pdf, png, tif/tiff) не конвертируем — только перемещаем.
- Все изменения оформляем через pull request с понятным описанием.
