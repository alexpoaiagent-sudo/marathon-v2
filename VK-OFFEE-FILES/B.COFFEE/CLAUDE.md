# Инструкции для Claude Code и AI-агентов

Этот документ содержит инструкции для работы с базой знаний сети «Вкусный Кофе».

## Структура хранилища

База организована по методу **SRT (Systems-Roles-Table)** — таблица 3×3 систем и ролей:

- **F0** — Метасемейство (управление хранилищем)
- **F1-F3** — Надсистема (рынок HoReCa)
- **F4-F6** — Целевая система (сеть кофеен)
- **F7-F9** — Система создания (команда управления)

## Три системы проекта

### System-of-Interest (Целевая система)
**Сеть кофеен «Вкусный Кофе»** — то, что создаём.

### Suprasystem (Надсистема)
**Городская среда и рынок HoReCa** — система, в которую встроены кофейни.

### Constructor (Система создания)
**Команда управления** — система, которая создаёт и развивает сеть.

## Правила работы с хранилищем

### 1. Навигация
- Всегда начинай с [content/README.md](content/README.md)
- Используй таблицу SRT 3×3 для понимания структуры
- Следуй по ссылкам между документами

### 2. Терминология
- Используй термины из [glossary.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/glossary.md)
- **P95** — 95-й перцентиль времени выдачи
- **AOV** — средний чек
- **CRR** — доля повторных визитов
- **NPS** — индекс лояльности
- **Фудкост** — себестоимость / выручка

### 3. Системное мышление
При анализе используй методы:
- **Чёрный ящик** — анализ входов/выходов без внутренних процессов
- **Прозрачный ящик** — анализ внутренних процессов
- **Холархия** — иерархия систем (надсистема → система → подсистемы)

### 4. Создание документов
При создании новых документов:
- Определи семейство (F0-F9)
- Используй YAML front matter:
  ```yaml
  ---
  title: "Название документа"
  section: "раздел"
  role: ["роль1", "роль2"]
  status: "green|yellow|red"
  owner: "Ответственный"
  last_reviewed: "YYYY-MM-DD"
  next_review: "YYYY-MM-DD"
  ---
  ```
- Добавь в [knowledge-inventory.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/knowledge-inventory.md)

### 5. Обновление документов
- Проверяй актуальность ссылок
- Обновляй `last_reviewed` и `next_review`
- Меняй `status` при необходимости (green/yellow/red)

## Типичные задачи

### Поиск информации
1. Проверь [knowledge-inventory.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/knowledge-inventory.md)
2. Используй структуру SRT для навигации
3. Читай README в каждом разделе

### Анализ процессов
1. Определи, к какой системе относится процесс
2. Найди соответствующее семейство (F0-F9)
3. Используй методы «чёрного» и «прозрачного» ящика

### Работа с метриками
- **Гостевой опыт:** P95, NPS, CRR
- **Экономика:** AOV, Фудкост, Выручка/час
- **Команда:** Часы/₽, Текучесть

## Семейства документов (F0-F9)

### F0: Метасемейство
- Глоссарий, инвентарь знаний
- Процессы работы с хранилищем
- Планы и отчёты

### F1: Контекст и рынок (Надсистема)
- Анализ рынка HoReCa

### F2: Окружение и интерфейсы (Надсистема)
- Инфраструктура районов

### F3: Партнёрства (Надсистема)
- Франчайзинг
- Партнёрские программы
- Работа с регуляторами

### F4: Ценности и описание (Целевая система)
- Системное описание сети
- Ценности кофейни
- Видение и миссия

### F5: Меню и продукты (Целевая система)
- Меню кофеен
- Составы продукции
- Рецептуры и стандарты

### F6: Операции и качество (Целевая система)
- Чек-листы
- Подсказки для бариста
- Контроль качества

### F7: Финансы и принципы (Система создания)
- Себестоимость и калькуляции

### F9: Команда и обучение (Система создания)
