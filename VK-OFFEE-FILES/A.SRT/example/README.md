# Planora — Хранилище знаний

Хранилище знаний проекта Planora, организованное по SRT-методу с двухъядерной структурой.

## Цепочка создания ценности

```
Planora → Приложение → Создатель ORA → ORA → Впечатлённый человек → Среда
```

## Два ядра хранилища

Хранилище организовано вокруг **двух системных ядер**:

| Ядро | Название | Целевая система | Надсистема | Система создания |
|------|----------|-----------------|------------|------------------|
| **A.Target** | Целевая система | Впечатлённый человек | Среда человека | ORA (ивент-продукт) |
| **B.Product** | Наша система | Приложение | Создатель ORA | Planora |

### Визуализация связей

```
┌─────────────────────────────────────────────────────────────────┐
│                         ЯДРО A (Target)                         │
│                                                                 │
│  Среда человека → Впечатлённый человек ← ORA (ивент-продукт)   │
│       (A1)              (A2)                  (A3)              │
│                                                │                │
│                                                │ использует     │
├──────────────────────────── X.Bridge ──────────┼────────────────┤
│                                                │                │
│                                                ▼                │
│                         ЯДРО B (Product)                        │
│                                                                 │
│    Создатель ORA    →    Приложение    ←    Planora            │
│        (B1)                (B2)              (B3)               │
└─────────────────────────────────────────────────────────────────┘
```

## Определения систем

### Ядро A: Целевая система

1. **Целевая система (A2)** — **Впечатлённый человек**
   Человек как система, которого ORA переводит в целевое состояние "впечатлённости" в контексте X и окне времени W.

2. **Надсистема (A1)** — **Среда человека**
   Устойчивые системы окружения: семья, круг общения, город, инфосреда.

3. **Система создания (A3)** — **ORA (ивент-продукт)**
   Мероприятие-как-продукт, воздействующее на человека и повышающее его впечатлённость.

### Ядро B: Наша система

4. **Целевая система (B2)** — **Приложение**
   Система, поддерживающая создание и проведение ORA: регистрация, билеты, коммуникации, данные.

5. **Надсистема (B1)** — **Создатель ORA**
   Система, которая проектирует и проводит ORA, используя Приложение как подсистему.

6. **Система создания (B3)** — **Planora**
   ИТ-служба (создаёт Приложение) + ORA-студия (внедряет в практику).

| Код | Система | Роль | Содержание |
|-----|---------|------|------------|
| A1.1 | Среда | Предприниматель | Зачем человеку мероприятия |
| A1.2 | Среда | Инженер | Структура среды: семья, город, инфосреда |
| A1.3 | Среда | Менеджер | Поведение человека в среде |
| A2.1 | Впечатлённый человек | Предприниматель | Что такое впечатлённость, критерии |
| A2.2 | Впечатлённый человек | Инженер | Компоненты: эмоции, связи, знания |
| A2.3 | Впечатлённый человек | Менеджер | Путь участника |
| A3.1 | ORA | Предприниматель | Философия создания мероприятий |
| A3.2 | ORA | Инженер | Структура ORA, описания |
| A3.3 | ORA | Менеджер | Процесс создания и проведения |

### Ядро B (Product) — 9 семейств

| Код | Система | Роль | Содержание |
|-----|---------|------|------------|
| B1.1 | Создатель ORA | Предприниматель | Ценность для организатора |
| B1.2 | Создатель ORA | Инженер | Структура: организатор + площадка + ... |
| B1.3 | Создатель ORA | Менеджер | Workflow создателя |
| B2.1 | Приложение | Предприниматель | Требования, User Stories |
| B2.2 | Приложение | Инженер | Компоненты, API, данные |
| B2.3 | Приложение | Менеджер | Гайды, сценарии использования |
| B3.1 | Planora | Предприниматель | Миссия, бизнес-модель |
| B3.2 | Planora | Инженер | ИТ-служба + ORA-студия |
| B3.3 | Planora | Менеджер | Команда, процессы |

## Быстрый старт

1. Изучите [определения систем](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/systems-definition.md)

   - **ИТ-служба** → Ядро B (Product)
3. Используйте [мосты](content/X.Bridge/) для понимания связей
