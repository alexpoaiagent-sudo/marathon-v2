## Особенность: два ядра

В отличие от стандартного SRT (одна таблица 3×3), Planora использует **двухъядерную структуру**:

```
0.Management          — общее управление
A.Target (9 семейств) — Впечатлённый человек
B.Product (9 семейств) — Приложение
X.Bridge              — связи между ядрами
```
