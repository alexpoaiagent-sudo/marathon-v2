# Раздел 0. Управление (Management)

## Назначение раздела

Раздел **Management** — это **метасистема хранилища** (F0), которая управляет логикой организации знаний, рабочими процессами и автоматизацией.

В отличие от F1-F9, метасемейство F0 **не делится по ролям** Предприниматель/Инженер/Менеджер. Вместо этого оно организовано по функциональным областям.

## Детали подразделов

### 0.1. Логика хранилища и знаний

**Ключевой вопрос:** Как организованы знания?

Фундаментальные основы организации хранилища: модель семейств документов, принципы, терминология. "Конституция" репозитория.

**Ключевые документы:**
- `document-families.md` — **главный документ**: модель семейств F0-F9
- `principles.md` — принципы организации
- `glossary.md` — глоссарий терминов

### 0.2. Процессы работы с хранилищем

**Ключевой вопрос:** Как работать с хранилищем?

Операционные процессы: структура папок, стандарты оформления, workflows создания документов.

**Ключевые документы:**
- `structure.md` — структура папок
- `standards.md` — стандарты оформления
- `document-creation.md` — процесс создания документов

### 0.3. Планы и совещания

**Ключевой вопрос:** Что запланировано?

Планирование развития хранилища: roadmap, протоколы совещаний, координация.

### 0.4. Автоматические отчёты ИИ

**Ключевой вопрос:** Какое состояние хранилища?

Автоматически генерируемые отчёты AI о качестве, покрытии, битых ссылках.

### 0.9. Входящие

**Ключевой вопрос:** Что нужно обработать?

Inbox для новых идей, черновиков и материалов до классификации по семействам.

### 0.99. Архив

**Ключевой вопрос:** Что было раньше?

Хранилище устаревших документов для сохранения истории.

## Принципы работы с разделом

1. **SSOT** — каждая сущность имеет одно место определения
2. **Метаданные обязательны** — YAML frontmatter для всех документов
3. **Разделение ролей** — люди: стратегия, AI: рутина
4. **Инкрементальное развитие** — не создавай документы "на будущее"
