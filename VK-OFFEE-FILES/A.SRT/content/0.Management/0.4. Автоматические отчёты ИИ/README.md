# 0.4. Автоматические отчёты ИИ

## Назначение раздела

Здесь размещаются **автоматически генерируемые отчёты** AI-агентов о состоянии хранилища, качестве документов и результатах проверок.

**Ключевой вопрос:** Что сделал AI? Какое состояние хранилища?

## Типы отчётов

| Тип отчёта | Описание | Периодичность |
|------------|----------|---------------|
| Качество документов | Проверка frontmatter, связей | По запросу |
| Покрытие таблицы | Заполненность SRT-ячеек | Еженедельно |
| Битые ссылки | Проверка related и внешних ссылок | По запросу |
| FPF-соответствие | Аудит на соответствие принципам | По запросу |
| **Системная идентификация** | Проверка соответствия систем правилам [system-identification.md](../0.1.%20Логика%20хранилища%20и%20знаний/system-identification.md) | По запросу |

## Обязательная проверка: Системная идентификация

> **ВАЖНО:** При генерации любого отчёта AI-агент **обязан** проверять соответствие описанных систем правилам из [system-identification.md](../0.1.%20Логика%20хранилища%20и%20знаний/system-identification.md).

### Что проверяется

1. **Целевая система (SoI):**
   - Физический объект или состояние (не процесс)
   - Можно измерить или "потрогать"
   - Не путается с создателем

2. **Надсистема (Suprasystem):**
   - Целевая система является её частью (composition)
   - Не путается с рынком или контекстом
   - Описан интерфейс взаимодействия

3. **Система создания (Constructor):**
   - Понятен состав: люди + инструменты + методы
   - Понятен результат работы

4. **Соответствие глоссарию:**
   - Терминология из [glossary.md](../0.1.%20Логика%20хранилища%20и%20знаний/glossary.md)
   - Системы соответствуют цепочке: `Planora → Приложение → Создатель ORA → ORA → Впечатлённый человек → Среда`

### Команда для проверки

```
Проверь документ [path] на соответствие system-identification.md:
- Правильно ли определена целевая система?
- Не путается ли надсистема с контекстом?
- Корректно ли описан создатель?
```

